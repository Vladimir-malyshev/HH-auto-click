# НАВЫК: hh_apply

## Название
`hh_apply`

## Описание
Автоматизирует отклики на вакансии на `hh.ru`. Содержит всю техническую реализацию процесса.

## Команды
- `apply_to_search(search_url)` — запустить отклики для страницы поиска.
- `run_diagnostics()` — проверить окружение. Только по явной команде пользователя, никогда автоматически.

## Основной процесс (apply_to_search)

⚠️ `execute_browser_javascript` использовать ТОЛЬКО на шагах 4б и 5в — и нигде больше.

Для **каждой** вакансии на странице:

### Шаг 1 — Проверка статуса
Визуально проверить блок вакансии. 
[cite_start]Никакого пролистывания[cite: 11]. [cite_start]Начинать строго с самого верха списка[cite: 12]. 
Если статус "Вы откликнулись" — пропустить, перейти к следующей. [cite_start]Если статуса нет — откликаться[cite: 13].

### Шаг 2 — Открыть форму
[cite_start]Делать строго **один клик** на кнопку "Откликнуться" в блоке вакансии[cite: 2].
Дождаться появления модального окна с формой. [cite_start]Повторные клики запрещены, так как они открывают лишние вкладки[cite: 1].
[cite_start]Если вкладка всё же открылась (редирект или случайный клик) — немедленно закрыть её, прежде чем идти дальше[cite: 4].
➡️ Только после того как модальное окно открылось — переходить к шагу 3.

### Шаг 3 — Выбрать резюме
Внутри открытой формы выбрать резюме:
- Название вакансии содержит Product, Manager, Owner, Владелец, Менеджер → "AI Product Manager"
- Иначе → "AI Product Leader"

### Шаг 4 — Поле зарплаты
Внутри открытой формы проверить: есть ли поле для ввода зарплаты.

а) Если поля нет — перейти к шагу 5.

б) Если поле есть — вставить `400000` через `execute_browser_javascript`:
```javascript
(function(value) {
  const input = document.querySelector('input[data-qa="salary-input"]') ||
                document.querySelector('input[type="number"]');
  if (!input) return 'NOT_FOUND';
  const nativeSetter = Object.getOwnPropertyDescriptor(
    window.HTMLInputElement.prototype, 'value'
  ).set;
  nativeSetter.call(input, value);
  input.dispatchEvent(new Event('input', { bubbles: true }));
  input.dispatchEvent(new Event('change', { bubbles: true }));
  return input.value;
})('400000')

Если вернулось 'NOT_FOUND' — поля нет, перейти к шагу 5.
Шаг 5 — Сопроводительное письмо

Внутри открытой формы проверить: есть ли под полем текст "Обязательное поле для этой вакансии".

а) Если НЕ обязательно — перейти к шагу 6.

б) Если обязательно:

    Определить файл: PM-роль → cover_letters/ai_product_manager.txt, иначе → cover_letters/ai_leader.txt

    Прочитать через read_file, сохранить в COVER_TEXT. Внимание: Открывать файлы в браузере категорически запрещено. Это строго фоновая операция чтения.

    Проверить COVER_TEXT: не пустой, длина > 100, не содержит test/hello/asdf/проверка/TODO

    Если проверка не прошла → записать ошибку в лог, закрыть форму, перейти к следующей вакансии

в) Вставить COVER_TEXT через execute_browser_javascript:
JavaScript

(function(text) {
  const tx = document.querySelector(
    'textarea[data-qa="vacancy-response-letter-informer"]'
  );
  if (!tx) return 'NOT_FOUND';
  const nativeSetter = Object.getOwnPropertyDescriptor(
    window.HTMLTextAreaElement.prototype, 'value'
  ).set;
  nativeSetter.call(tx, text);
  tx.dispatchEvent(new Event('input', { bubbles: true }));
  tx.dispatchEvent(new Event('change', { bubbles: true }));
  return tx.value.length;
})(COVER_TEXT)

⚠️ COVER_TEXT подставлять как JS-строку: кавычки → \", переносы → \n.

г) Проверить результат: число > 100 и счётчик не "0из10000" → OK.
Если 0 или 'NOT_FOUND' → записать ошибку в лог, перейти к следующей вакансии.
Шаг 6 — Отправка

Нажать кнопку "Откликнуться" внутри формы.
Дождаться подтверждения (кнопка изменилась или появилось сообщение об успехе).
Шаг 7 — Уборка и логирование

    Закрыть все вкладки кроме страницы поиска.

    Записать в applications.log обязательно:

        Успех: [Дата время] | [Вакансия] | Резюме: [название] | Письмо: Да/Нет | Статус: Успех

        Ошибка: [Дата время] | ОШИБКА | Инструмент: [имя] | Причина: [текст] | Вакансия: [название]

Шаг 8 — Следующая вакансия

Если вакансии на странице закончились → нажать следующую страницу (a[data-qa="pager-next"]).
Если страницы кончились → завершить, вывести итоговую статистику из лога.
КРИТИЧЕСКИЙ ЗАПРЕТ

    ЗАПРЕЩЕНО вызывать execute_browser_javascript до открытия модальной формы отклика.

    ЗАПРЕЩЕНО вставлять текст через browser_press_key, type или буфер обмена.

    ЗАПРЕЩЕНО использовать javascript: в адресной строке браузера.

    ЗАПРЕЩЕНО нажимать "Откликнуться" если счётчик письма показывает "0из10000".

    ЗАПРЕЩЕНО импровизировать — при ошибке записать в лог и идти дальше.

    ЗАПРЕЩЕНО скроллить страницу или использовать поиск по странице для возврата к старым вакансиям.

Анти-детект

Рандомизировать задержки 2–5 сек между действиями. Имитировать естественную прокрутку.
