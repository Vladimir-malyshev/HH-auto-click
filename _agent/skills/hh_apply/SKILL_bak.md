# НАВЫК: hh_apply

## Название
`hh_apply`

## Описание
Автоматизирует отклики на вакансии на `hh.ru`, используя сохраненные поиски и постоянную сессию браузера.

## Команды
- `apply_to_search(search_url)`: Запускает процесс откликов для конкретной страницы результатов поиска.

## Детали реализации
- Использует встроенный **Antigravity Browser Subagent**.
- Процедура для КАЖДОЙ вакансии в списке:
    1.  Найти кнопку "Откликнуться" в блоке вакансии.
    2.  Проверить, нет ли статуса "Вы откликнулись". Если есть — пропустить.
    3.  Нажать "Откликнуться" прямо в списке (без перехода на страницу вакансии).
    4.  В появившейся форме выбора резюме:
        - Проверить название вакансии на ключевые слова PM-ролей (Product, Manager, Владелец проекта).
        - Если PM: выбрать "AI Product Manager".
        - Иначе: выбрать "AI Product Leader".
    5.  Проверить, является ли сопроводительное письмо ОБЯЗАТЕЛЬНЫМ (текст "Сопроводительное письмо" и ниже "обязательное поле для этой вакансии").
        - Если обязательно: взять текст из соответствующего файла (`cover_letters/*.txt`).
        - **КРИТИЧЕСКИЙ ЗАПРЕТ (STOP RULES)**:
            *   ЗАПРЕЩЕНО использовать любые тестовые или placeholder тексты ("test", "hello", "asdf", "проверка"). Только полные тексты из файлов.
            *   ЗАПРЕЩЕНО использовать `browser_press_key` или `type` для ввода кириллицы.
            *   ЗАПРЕЩЕНО использовать `javascript:` в строке адреса (URL).
        - **ТЕХНИЧЕСКАЯ РЕАЛИЗАЦИЯ ВСТАВКИ**:
            1. Найти textarea: `textarea[data-qa="vacancy-response-letter-informer"]`.
            2. Вставить текст через `execute_browser_javascript`:
               ```javascript
               (function(text) {
                 const tx = document.querySelector('textarea[data-qa="vacancy-response-letter-informer"]');
                 if (tx) {
                   tx.value = text;
                   tx.dispatchEvent(new Event('input', { bubbles: true }));
                   tx.dispatchEvent(new Event('change', { bubbles: true }));
                   return true;
                 }
                 return false;
               })("ПОЛНЫЙ_ТЕКСТ_ПИСЬМА")
               ```
            3. Обязательно проверить, что текст появился в поле, прежде чем нажимать "Откликнуться".
        - Если не обязательно: отправить без письма.
    6.  Нажать "Откликнуться".
    7.  Подтвердить успех (проверить изменение кнопки или сообщение об успехе).
    8.  **БЕЗОПАСНОСТЬ ПАМЯТИ**: После каждого отклика или возврата к поиску принудительно проверять список вкладок и закрывать все, кроме основной страницы поиска. Это критично для стабильности браузера.
    9.  **ВАЖНО**: Если после отклика (особенно если была анкета) вы покинули страницу поиска, необходимо просто заново открыть исходную ссылку на поиск — обработанные вакансии исчезнут из выдачи автоматически.
- **Пагинация**: Когда вакансии на странице закончились, прокрутить вниз и нажать на следующую цифру страницы (например, если текущая 1, нажать на 2) или на кнопку-стрелку "Дальше" (селектор `a[data-qa="pager-next"]`). Повторять до завершения всех доступных страниц.
- Рандомизировать задержки и прокрутку для имитации поведения человека.
- Записывать результаты в `applications.log`.
