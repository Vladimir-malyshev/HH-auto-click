# Правила для HH Auto-Click

## Основная цель
Автоматизация откликов на вакансии на HeadHunter (hh.ru) с использованием сохраненных поисков и готовых резюме.

## Языковые правила
Все общение, планы, отчёты и комментарии вести исключительно на **русском языке**.

## Правила выбора резюме
- Для ролей с ключевыми словами в названии (Product, Manager, Owner, Владелец, Менеджер) → резюме **"AI Product Manager"**
- Для всех остальных → резюме **"AI Product Leader"**

## Браузерная сессия
- Всегда использовать встроенный браузер с сохранённой авторизацией.
- Если сессия истекла — уведомить пользователя и дождаться ручного входа.

## Процесс отклика

### Основной порядок действий
1. Найти кнопку "Откликнуться" в блоке вакансии.
2. Проверить статус — если "Вы откликнулись", пропустить.
3. Нажать "Откликнуться" прямо в списке (без перехода на страницу вакансии).
4. Выбрать резюме согласно правилам выше.
5. Обработать поле сопроводительного письма (см. раздел ниже).
6. Нажать "Откликнуться".
7. Подтвердить успех (кнопка изменилась или появилось сообщение).

### Сопроводительное письмо

Письмо отправлять **ТОЛЬКО** если в форме есть текст "Обязательное поле для этой вакансии".

**Если обязательно — строго следовать порядку:**

а) Определить файл:
   - PM-роль → `cover_letters/ai_product_manager.txt`
   - Иначе → `cover_letters/ai_leader.txt`

б) Прочитать файл через `read_file`, сохранить в переменную COVER_TEXT.

в) Проверить COVER_TEXT:
   - Не пустой
   - Длина > 100 символов
   - Не содержит: test, hello, asdf, проверка, TODO, placeholder
   Если проверка не прошла — СТОП, записать ошибку в лог, перейти к следующей вакансии.

г) Вставить через `execute_browser_javascript`:
   ```javascript
   (function(text) {
     const tx = document.querySelector(
       'textarea[data-qa="vacancy-response-letter-informer"]'
     );
     if (!tx) return 'NOT_FOUND';
     const nativeSetter = Object.getOwnPropertyDescriptor(
       window.HTMLTextAreaElement.prototype, 'value'
     ).set;
     nativeSetter.call(tx, text);
     tx.dispatchEvent(new Event('input', { bubbles: true }));
     tx.dispatchEvent(new Event('change', { bubbles: true }));
     return tx.value.length;
   })(COVER_TEXT)
   ```
   ⚠️ COVER_TEXT подставляется как JavaScript-строка с экранированием кавычек (`\"`) и переносов (`\n`).

д) Проверить результат: функция должна вернуть число > 100.
   - Если `0` или `'NOT_FOUND'` — СТОП, записать ошибку в лог, перейти к следующей вакансии.
   - Если успех — счётчик символов в поле должен измениться (не "0из10000").

**Если не обязательно:** пропустить поле, перейти к следующему шагу.

### КРИТИЧЕСКИЙ ЗАПРЕТ (STOP RULES)
- ЗАПРЕЩЕНО вставлять тестовые или placeholder тексты ("test", "hello", "asdf", "проверка"). Только тексты из файлов.
- ЗАПРЕЩЕНО использовать `browser_press_key` или `type` для ввода кириллицы.
- ЗАПРЕЩЕНО использовать `javascript:` в строке адреса браузера.
- ЗАПРЕЩЕНО нажимать "Откликнуться" если счётчик символов показывает "0из10000".

## Управление вкладками
После каждого отклика принудительно проверять список вкладок и закрывать все, кроме основной страницы поиска. Критично для стабильности браузера.

## Пагинация
После обработки всех вакансий на странице — нажать следующую цифру страницы или кнопку "Дальше" (селектор `a[data-qa="pager-next"]`). Повторять до завершения всех страниц.

## Анти-детект
- Рандомизировать задержки между действиями (2–5 секунд).
- Имитировать естественную прокрутку страницы.

## Логирование
Вести лог в `applications.log`. Запись ОБЯЗАТЕЛЬНА после каждого действия. Без записи в лог действие считается незавершённым.

Форматы записей:
- Успех: `[Дата время] | [Вакансия] | Резюме: [название] | Письмо: Да/Нет | Статус: Успех`
- Ошибка: `[Дата время] | ОШИБКА | Инструмент: [имя] | Причина: [текст] | Вакансия: [название]`
